## Golden Tasks
source: ultraflow/SKILL.md#evals.yaml + ultraflow/eval-report.json

- ultraflow has 5 golden tasks in evals.yaml, one per intent shape plus the ground-halt: route-decision-no-forge, route-build-skips-forge, route-forge-full-motion, ground-halt-ungrounded, route-govern-no-build.
- Each task is a prompt + must[] + must_not[] of OBSERVABLE strings the output contract emits. Scored on literal substring presence, never on "reads well".
- must[] entries target the four required_sections: `## ROUTE`, `Shape: forge`, `## GROUNDED`, `Verdict: NO`, `## FORGE REPORT`, `Forge: skipped`, `## LANE EARNED`, `Lane: authored`, `tfc_pack_bridge`.
- must_not[] serves double duty as a voice gate: `crucial`, `delve`, `robust`, `nuanced`, `multifaceted`, `comprehensive` are all banned words that signal LLM filler. A passing run also proves clean voice.
- Minimum 3 tasks must pass to reach eval_proven lane (pass_threshold: 0.8 over 5 tasks = 4 required, but 5 tasks x 0.8 = 4.0 so 4+ must pass).
- The ground-halt task (`ground-halt-ungrounded`) is the most critical: it verifies that an ungrounded intent emits `Verdict: NO` and `HALTED` without ever reaching `Shape: forge` or any Forge path.

## Judge Rubrics
source: ultraflow/SKILL.md#the-output-contract + ultraflow/eval-report.json

- The judge is model-free where possible: run the task baseline (no skill loaded) then skill-loaded, grep output for each must/must_not string. Deterministic, repeatable, no API key (INV-1).
- A task's `pass` is `true` IFF every `must` matched AND no `must_not` matched. Otherwise `false`. No partial credit.
- `behavioral_score` = (count of tasks with pass:true) / (total tasks judged). A float 0..1. Do not round kindly.
- `delta_note` must state the OBSERVABLE difference: what the baseline missed that the skill-loaded run produced. If the skill-loaded run failed a check, name exactly which string was missing or forbidden.
- lane-gate.sh validates the report: rejects if behavioral_score < pass_threshold, if `per_task` count != total tasks, if any task is missing must/must_not, or if skill_version doesn't match the spec.
- A skills-loaded run that scores the SAME as baseline gets a low score -- that is the eval-theater firewall (V3). The skill must change observable output to earn a lane.

## Regression Signals
source: ultraflow/SKILL.md#sharp-edges + ultraflow/spec.yaml#freshness_horizon

- evalStale: spec version moved past the eval-report version. tfc_lane drops the lane to authored silently until a fresh report lands. Re-eval on every version bump.
- A dropped `required_section` fails tfc_behavioral before it ships -- that is the gate that stops output quality from scaling only with the running agent's capability (v3 W3 behavioral invariant).
- decayPressure: a proof older than the freshness_horizon drops the effective lane: eval_proven decays after 30 days, evolution_proven after 60 days. Check via tfc_portfolio or tfc_lane.
- voice-gate-managed-block-false-positive: tfc_validate's voice/path gates fire on the managed TFC:PREAMBLE-HOOK block. These are regenerated by tfc_install; the author cannot fix them. They dock rubric points honestly -- document, do not patch.
- A behavioral_score of 1.0 means the skill cannot earn evolution_proven via the delta gate (needs delta >= 0.05 over pre_eval_score). When pre_eval_score is already 1.0, evolve writes an honest CHANGELOG row with delta 0.0 and stays eval_proven.
